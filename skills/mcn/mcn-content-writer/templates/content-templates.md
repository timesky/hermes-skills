# 公众号文章改写模板库

针对不同文章类型和风格的 prompt 模板，确保内容多样性和去 AI 化效果。

---

## 一、文章类型分类

| 类型 | 特点 | 适用场景 | 模板编号 |
|------|------|----------|----------|
| **热点评论** | 快速响应、观点鲜明 | 社会事件、科技新闻 | T-01 |
| **干货教程** | 实操性强、步骤清晰 | 技术分享、工具使用 | T-02 |
| **行业分析** | 数据支撑、深度解读 | 市场趋势、公司动态 | T-03 |
| **故事叙述** | 情感共鸣、代入感强 | 个人经历、创业故事 | T-04 |
| **观点争议** | 引发讨论、独特视角 | 行业争议、技术选型 | T-05 |

---

## 二、改写风格定义

### Style A: professional（专业严谨）

**特征**：
- 客观分析，数据支撑
- 专业术语适度使用
- 结构清晰，逻辑严密
- 适合：行业分析、技术深度文

**去 AI 化要点**：
- 减少"作为..."、"此外"等套话
- 用行业从业者视角，而非旁观者
- 添加个人判断："我认为"、"从我的经验来看"

---

### Style B: casual（轻松口语）

**特征**：
- 口语化表达，接地气
- 段落短小，节奏快
- 适度调侃，但不失专业
- 适合：干货教程、热点评论

**去 AI 化要点**：
- 用"说实话"、"你想想"等口语
- 打破三段式列举
- 用比喻、场景化描述

---

### Style C: story（故事叙述）

**特征**：
- 时间线推进
- 人物/场景描写
- 情感起伏，有张力
- 适合：创业故事、产品历程

**去 AI 化要点**：
- 用第一人称或"他"叙事
- 添加内心独白
- 避免"总结来说"，用场景收尾

## 三、Prompt 模板

### T-01：热点评论模板

```markdown
# 任务：热点评论改写

## 输入
- 原标题：{original_title}
- 原文摘要：{original_summary}
- 核心事件：{core_event}
- 参考原文：{reference_article}

## 改写要求

### 结构框架（5段式）
1. **开篇引入**（150字）：用场景或反差切入，引发好奇
2. **事件还原**（300字）：简述事件核心，不啰嗦
3. **观点分析**（500字）：2-3个分析角度，有数据支撑
4. **延伸思考**（400字）：关联行业/社会，升华价值
5. **结尾互动**（150字）：提问引导，不喊口号

### 写作风格：{style}

### 去 AI 化要求
- ❌ 禁止："作为...的证明"、"综上所述"、"值得注意的是"
- ✅ 使用："说实话"、"我觉得"、"你想想"、"有意思的是"
- ✅ 添加个人视角：程序员/从业者/观察者的真实感受
- ✅ 句子节奏变化：长短句交替，避免全是陈述句

### 字数要求
- 总字数：1500-2000字
- 开篇不超过150字，结尾不超过150字

## 输出格式
直接输出文章正文，无需标题（标题单独生成）
```

### T-02：干货教程模板

```markdown
# 任务：干货教程改写

## 输入
- 教程主题：{topic}
- 核心知识点：{key_points}
- 目标读者：{target_audience}
- 参考原文：{reference_article}

## 改写要求

### 结构框架（步骤式）
1. **痛点引入**（100字）：读者遇到的实际问题
2. **方案概述**（200字）：本文要解决什么，为什么有效
3. **步骤详解**（800字）：3-5个实操步骤，每步有示例
4. **进阶技巧**（400字）：常见坑点、优化建议
5. **总结回顾**（150字）：核心要点速记

### 写作风格：casual（推荐）

### 去 AI 化要求
- ❌ 禁止："首先...其次...最后..."的机械列举
- ✅ 使用："先说说...然后是...最后别忘了..."
- ✅ 添加真实经验："我之前踩过坑..."、"实测发现..."
- ✅ 用比喻：技术概念用生活比喻解释

### 字数要求
- 总字数：1500-2000字
- 步骤部分占比最大

## 输出格式
直接输出文章正文
```

### T-03：行业分析模板

```markdown
# 任务：行业分析改写

## 输入
- 分析主题：{topic}
- 涉及公司/产品：{entities}
- 市场数据：{market_data}
- 参考原文：{reference_article}

## 改写要求

### 结构框架（分析式）
1. **现象引入**（150字）：行业动态/新闻事件切入
2. **数据呈现**（300字）：市场规模、增长率、关键数字
3. **深度分析**（600字）：2-3个分析维度（竞争格局、技术趋势、商业模式）
4. **未来预判**（400字）：基于分析的合理预测
5. **读者启示**（150字）：对从业者/投资者的实际意义

### 写作风格：professional（推荐）

### 去 AI 化要求
- ❌ 禁止：纯数据罗列无观点
- ✅ 使用："这背后反映的是..."、"从数据能看出..."
- ✅ 添加个人判断："我认为这个趋势..."、"值得关注的是..."
- ✅ 避免过于正式：适度口语化，不像研究报告

### 字数要求
- 总字数：1500-2000字

## 输出格式
直接输出文章正文
```

## 四、标题优化生成流程（2026-04-16 更新）

### 步骤 1：提取核心关键词

从改写后的原文提取 **5个核心关键词**：

| 选择标准 | 说明 |
|----------|------|
| 话题相关性 | 最能代表文章主题的词 |
| 搜索热度 | 热搜词、高频搜索词 |
| 品牌提及 | 涉及的公司/产品名称 |
| 技术概念 | 文章讨论的核心技术 |
| 独特性 | 能区分文章的差异化词 |

**提取原则**：
- 必须在原文中出现且有实际意义
- 避免泛泛词汇（如"手机"、"科技"）
- 优先选择热搜榜上的词

### 步骤 2：生成备选标题

结合以下元素生成 **5个备选标题**（每个15-25字）：

1. **原标题（主题）**
2. **5个核心关键词**
3. **该分类热搜词**

**标题公式库**：

| 公式 | 结构 | 示例 | 适用场景 |
|------|------|------|----------|
| A | 热点词 + 疑问 + 结论提示 | 折叠iPhone为何难产？拆了卷轴屏后我懂了 | 热点评论 |
| B | 品牌 + 热点 + 真相揭秘 | 苹果折叠屏为何迟到？LG卷轴屏揭秘真相 | 行业分析 |
| C | 热点事件 + 背后 + 深度分析 | 折叠iPhone试产受挫背后：卷轴屏揭示柔性屏困境 | 深度解读 |
| D | 动作感 + 品牌 + 核心观点 | 拆解LG卷轴屏后明白：苹果折叠屏为何总比别人慢 | 技术分析 |
| E | 对比冲突 + 悬念 | 卷轴屏手机已死，折叠iPhone却还在路上：为什么？ | 观点争议 |

### 步骤 3：事实校验（新增，必做）

**⚠️ 重要**：标题生成前必须校验事实准确性，避免 AI 模型"幻觉"。

| 校验项 | 方法 |
|--------|------|
| 时间线 | 检查涉及产品/公司的实际发布时间（如 OpenClaw 2026年才火，不能写"用了两年"） |
| 数据准确性 | GitHub Stars、发布日期等数字需与实际数据一致 |
| 逻辑合理性 | 标题表达的经历/时长是否符合现实 |

**错误案例（2026-04-16）**：
- ❌ "Hermes 换掉用了两年的 OpenClaw" —— 事实错误！OpenClaw 和 Hermes 都是 2026 年才火起来的框架，不可能"用了两年"
- ✅ "从 OpenClaw 切换到 Hermes，省了八成 Token" —— 修正后，去掉时间表述

**校验方式**：
- 标题涉及产品 → 查 GitHub 创建时间/热度时间线
- 标题涉及公司 → 查公司成立时间/产品发布时间
- 标题涉及时长 → 确保时长表述符合现实

---

### 步骤 4：最优标题分析（四维度评估）

对5个备选标题进行四维度评分（每维度5分）：

| 维度 | 评分标准 | 高分特征 |
|------|----------|----------|
| **用户吸引力** | 能否最大化吸引点击（不夸张） | 悬念感、代入感、完整叙事弧、信息密度高 |
| **SEO友好度** | 关键词密度、搜索匹配度 | 包含热搜词、关键词自然嵌入、长尾词覆盖 |
| **热搜推荐潜力** | 能否被算法推荐 | 符合爆款模式、新闻时效性、情绪词适度 |
| **主题契合度** | 是否准确反映文章内容 | 与内容高度一致、无标题党、观点准确 |

**最优标题选择**：
- 总分最高者（≥18分）
- 若总分相同，优先"主题契合度"最高的
- 若仍有争议，选择"用户吸引力"最高的

### 实际案例（2026-04-16）

**原文关键词提取**：
1. 折叠iPhone（文章核心话题）
2. 苹果折叠屏（热搜词）
3. 卷轴屏手机（对比参照）
4. LG Rollable（具体案例）
5. 柔性屏（技术概念）

**备选标题**：
1. 折叠iPhone为何难产？拆了卷轴屏手机后我懂了
2. 苹果折叠屏为何迟到？LG卷轴屏手机揭秘真相
3. 折叠iPhone试产受挫背后：卷轴屏揭示柔性屏困境
4. 拆解LG卷轴屏后明白：苹果折叠屏为何总比别人慢
5. 卷轴屏手机已死，折叠iPhone却还在路上：为什么？

**最优标题分析**（标题1）：
- 用户吸引力 ★★★★★（悬念+完整叙事弧）
- SEO友好度 ★★★★★（折叠iPhone+难产+卷轴屏）
- 热搜推荐潜力 ★★★★☆（符合爆款模式，略缺情绪词）
- 主题契合度 ★★★★★（准确反映深度分析视角）
- **总分：19分 → 最优选择**

## 五、使用方法

### 在内容改写流程中调用

```python
def get_template(article_type: str, style: str) -> str:
    """获取对应模板"""
    
    templates = {
        '热点评论': 'T-01',
        '干货教程': 'T-02',
        '行业分析': 'T-03',
        '故事叙述': 'T-04',
        '观点争议': 'T-05'
    }
    
    template_id = templates.get(article_type, 'T-01')
    
    # 读取模板文件
    with open(f'templates/{template_id}.md', 'r') as f:
        template = f.read()
    
    # 替换变量
    return template.format(
        original_title=...,
        style=style,
        ...
    )
```

### 自动识别文章类型

```python
def detect_article_type(topic: str, source_content: str) -> str:
    """根据话题和内容自动识别类型"""
    
    # 关键词特征
    if any(kw in topic for kw in ['教程', '方法', '如何', '步骤']):
        return '干货教程'
    
    if any(kw in topic for kw in ['分析', '市场', '趋势', '估值']):
        return '行业分析'
    
    if any(kw in topic for kw in ['争议', '质疑', '问题', '危机']):
        return '观点争议'
    
    if any(kw in source_content for kw in ['经历', '故事', '创业', '历程']):
        return '故事叙述'
    
    # 默认热点评论
    return '热点评论'
```

## 六、Pitfalls

1. **模板不是万能**：根据具体话题灵活调整，不生搬硬套
2. **去 AI 化是核心**：模板只是结构，最终要有"人味"
3. **字数必须达标**：模板规定了字数分布，但总字数必须 1500-2000
4. **标题单独生成**：模板只针对正文，标题用公式另生成
5. **原文真实性**：改写必须基于真实原文，不能凭空捏造
